# DECISIONS

## D1: 折叠状态只放前端内存

服务组折叠是单页工作台的浏览体验，不属于排障 case 状态。当前只用前端 `Set` 保存展开状态，刷新后恢复默认展开，避免引入额外配置和迁移。

## D2: 图片预览复用原 file input

粘贴图片和手动选择图片仍统一写回 `#images.files`，提交时继续走已有 multipart `images` 字段。预览只读取待提交的 `FileList`，不新增临时上传 API。

## D3: 缩略图 URL 提交后主动释放

前端使用 `URL.createObjectURL` 生成本地缩略图，并在重新渲染、清空图片和离开页面时释放 URL，避免长时间工作台会话积累浏览器内存。
