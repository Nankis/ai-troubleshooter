# P-2026-030 Web Session Management

## 背景

Web 工作台左侧“问题”列表已有草稿和历史 case，但草稿只存在内存，刷新后丢失；问题会话也不能重命名或删除。用户希望这里支持删除和重命名，并明确会话内容到底放本地磁盘还是 DB。

## 目标

- 草稿问题支持重命名、删除和浏览器本地持久化。
- 已创建 case 支持重命名和软删除。
- 正式排查会话继续写 MySQL，保留审计、RAG、经验沉淀和多端恢复的事实源。
- Web UI 不把图片草稿持久化，避免本地长时间保留截图敏感信息。

## 非目标

- 不做文件系统会话仓库。
- 不把草稿同步到后端。
- 不实现复杂归档/恢复站。

## 验收标准

- `make test`、`make secret-scan`、`git diff --check` 通过。
- MySQL migration 可执行。
- Web 上草稿刷新后仍在，支持重命名/删除。
- 已创建 case 支持重命名/删除，删除后不再出现在问题列表。
