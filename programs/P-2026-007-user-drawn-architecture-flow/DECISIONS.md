# DECISIONS

## D1：用户手绘图作为当前架构图主事实源

README 和 ADR 的主图按用户手绘边界重画：Agent 平台、Investigation Gateway、业务服务能力和业务 DB 分层表达。

## D2：平台经验优先，不盲查下游

字段足够后，决策层先查平台经验并做置信度评分。只有低置信、证据不足或需要实时生产状态时，再通过 Gateway 查询业务证据。

## D3：高置信直接返回必须可审计

经验命中后可以直接返回，但必须记录经验来源、置信度、适用边界和未继续查下游的理由。

## D4：业务能力注册到 Gateway，业务服务自己访问 DB

Agent 不直接访问业务 DB、日志 MCP 或业务服务。Gateway 管只读能力注册和平台侧鉴权，业务服务再校验 Gateway 内部身份。

## D5：回复出口统一

决策层不直接给 Lark/飞书/Web 发消息，结论统一写回 Case API，再由 Channel Adapter 发送，保证消息和状态可审计。
